anna-app APS Files Demo

Saves text notes to Anna object storage two different ways — pick one with the toggle below:

  • Tool invoke — the app calls anna.tools.invoke against a bundled Executa that owns the aps.files capability and does the files/* reverse-RPC. The app needs no files grant. Writes to scope=user. (Recommended best practice.)
  • HOST API — the app calls anna.files.* directly with its own ui.host_api.files grant: upload_init → browser PUT to a presigned R2 URL → upload_finalize. Writes to scope=app (the app's own object space).

Run with anna-app dev --storage aps (real account). Without it, saves return a clean not_implemented (-32004). Note: the two modes use different scopes (user vs app), so a note saved in one mode is not listed by the other. HOST API uploads also require the storage bucket to allow CORS from this origin (configured in production).
